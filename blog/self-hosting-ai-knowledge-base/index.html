Self-Hosting Your AI Knowledge Base: A Complete Guide

Step-by-step guide to deploying Knowledge Nexus on your own infrastructure with Docker.

Why Self-Host?

Self-hosting gives you complete control over your data and infrastructure. Your documents never leave your servers, you choose your own LLM providers, and you can customize everything to your needs.

Benefits of Self-Hosting

  • ✓ Complete data sovereignty
  • ✓ Compliance with strict regulations (HIPAA, GDPR, etc.)
  • ✓ Customizable LLM providers (Ollama, vLLM, OpenAI, Anthropic)
  • ✓ Air-gapped deployment option
  • ✓ No per-query costs after initial setup

Prerequisites

  • Docker and Docker Compose installed
  • Minimum 8GB RAM (16GB+ recommended)
  • 50GB+ storage for documents and embeddings
  • Optional: NVIDIA GPU for local LLM inference

Quick Start

# Clone the repository
$ git clone https://github.com/knowledgenexus/knowledge-nexus.git
$ cd knowledge-nexus
# Copy environment template
$ cp .env.example .env
# Start all services
$ docker-compose up -d
# Access at http://localhost:8080

Architecture Overview

Knowledge Nexus consists of several microservices:

API Gateway

Routes requests, handles authentication, rate limiting

Document Processing

Parses PDFs, extracts text, generates embeddings

Vector Database (Qdrant)

Stores and searches document embeddings

LLM Service

Handles chat completions, can use local or cloud LLMs

Configuring Your LLM

Knowledge Nexus supports multiple LLM backends:

Option 1: Ollama (Local, Free)

LLM_PROVIDER=ollama
OLLAMA_BASE_URL=http://ollama:11434
OLLAMA_MODEL=llama3.2

Option 2: OpenAI API

LLM_PROVIDER=openai
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4-turbo

Option 3: Anthropic Claude

LLM_PROVIDER=anthropic
ANTHROPIC_API_KEY=sk-ant-...
ANTHROPIC_MODEL=claude-3-sonnet

Production Considerations

  • SSL/TLS: Use a reverse proxy (nginx, Traefik) with Let's Encrypt
  • Backups: Schedule regular backups of PostgreSQL and Qdrant data
  • Monitoring: Enable Prometheus metrics endpoint
  • Scaling: Run multiple replicas behind a load balancer
